### WorldGrid

главный класс владелец сетки. хранит какие клетки вообще есть (`valid_cells`) и какие из них заняты (`occupied_cells`)

- `world_to_cell(world_pos)` - мировая позиция -> координаты клетки
- `cell_to_world(cell_pos)` - обратная функция
- `get_overlap(rect)` - список занятых или несуществующих клеток в данном прямоугольнике
- `get_overlap_with_clearance(rect, clearance)` - то же самое но учитывая нужное между зданиями пространство
- `try_place_building(building)` - пытается разместить здание, если получилось - обновляет `occupied_cells` и возвращает `true`, иначе `false`
- `get_building_at_cell(cell)` - возвращает `name` того здания что занимает данную клетку или `""` если клетка пуста (не включает в себя `clearance`)

### GridRegion

класс для удобного хранения разрешенных областей сетки, знает свою позицию и размер. при запуске `WorldGrid` создает общий список разрешенных клеток на основе `GridRegion`ов-детей

- `contains(cell)` - есть ли в регионе данная клетка
- `get_cells()` - все клетки в регионе

### BuildingDefinition

ресурс шаблона здания. содержит название, описание, размер, раскладку вводов и выводов, цены на покупку и продажу, уровень здания и ресурс шаблона, в который здание апгрейдится, при наличии

- `get_building_instance()` - инстанцирует сцену здания, заполняет нужные поля и возвращает

### Building

базовый класс зданий на карте. содержит размер, позицию на сетке и требуемый от себя отступ

- `get_port(at, from)` - возвращает порт в клетке `at`, если идти к ней из клетки `from`. таким образом у нас может быть два порта на одной клетке в углу здания, если понадобится

### чуть-чуть про систему тиков и порядок действия построек

тик - глобальный счетчик времени. один тик *(предварительно, можете предлагать другие варианты)* длится 1/30 секунды или два кадра физики. каждый тик, все здания на карте исполняют одну из функций `tick_produce()`, `tick_transport()`, `tick_consume()`, синхронно, в цикле, бесконечно, именно в таком порядке.

```py
# в каком-нибудь условном world_grid.gd, не в самих зданиях
var tick_step := 0

func _physics_process(_delta):
    if Engine.get_physics_frames() % 2:
        return

    if tick_step == 0:
        for building in buildings:
            building.tick_produce()

    elif tick_step == 1:
        for building in buildings:
            building.tick_transport()

    elif tick_step == 2:
        for building in buildings:
            building.tick_consume()

    tick_step = (tick_step + 1) % 3
```

---

- `tick_produce()` - функция для генерации ресурсов. здесь условная ферма добавляет в `ItemInventory` своего выходного `BuildingPort`а 1 единицу чего бы там ни было

- `tick_transport()` - функция для перемещения ресурсов, в теории только для труб

- `tick_consume()` - функция для потребления ресурсов. здесь условная кухня проверяет входные `BuildingPort`ы на наличие нужных ресурсов, если они есть - убирает их и ставит себе флаг произвести еду на следующем `tick_produce()`
